# Obsidian Librarian 플러그인 PRD

## 프로젝트 핵심 정보

**목적**: Obsidian 볼트에 축적된 수백 개의 노트를 AI가 자동으로 요약/분류하고 태그를 추천하며, 토큰 효율적인 요약 인덱스를 활용한 자연어 질의로 규모에 관계없이 관련 노트를 즉시 찾을 수 있게 하여 정보 과부하와 노트 유실 문제를 해결한다.
**사용자**: 대규모 Obsidian 볼트를 운영하며 노트 정리와 재탐색에 어려움을 겪는 지식 노동자 및 연구자.

---

## 1. 프로젝트 개요 및 목표

### 핵심 문제

Obsidian 사용자는 노트를 빠르게 작성하지만 시간이 지나면서 노트가 수백~수천 개로 늘어난다. 수동으로 태그를 붙이고 폴더를 분류하는 작업은 번거롭고 일관성이 없으며, 나중에 필요한 노트를 찾을 때 키워드가 기억나지 않으면 관련 노트를 발견하지 못한다. 결과적으로 잘 작성된 노트가 묻혀버리고 지식이 재활용되지 못한다.

또한 기존의 전체 본문 전송 방식은 Claude API의 컨텍스트 한계(200K 토큰)와 비용 문제로 인해 300개 이상의 노트를 가진 볼트에서는 현실적으로 동작하지 않는다. 이 플러그인은 각 노트의 1~2문장 요약을 생성하고 이를 별도로 인덱싱하여 (800개 노트 기준 약 24K 토큰), 수천 개의 노트를 가진 볼트에서도 효율적으로 검색할 수 있게 한다.

### 비전

Obsidian Librarian은 볼트의 개인 사서 역할을 한다. 플러그인이 볼트를 스캔하여 AI로 각 노트의 내용을 분석하고, 1~2문장 요약과 함께 적절한 태그/분류를 추천한다. 이후 검색 시에는 전체 본문 대신 이 요약 데이터를 활용하여 토큰 한계 없이 수천 개의 노트를 효율적으로 탐색한다. 사용자는 추천을 검토하고 원하는 것만 선택 적용하여 볼트에 대한 통제권을 유지한다.

### 목표

- 노트 분류와 태그 작업에 드는 수동 시간을 80% 이상 절감한다.
- 요약 기반 인덱스로 볼트 규모(노트 1,000개 이상)에 관계없이 자연어 검색을 안정적으로 제공하고, API 비용을 전체 본문 전송 대비 90% 이상 절감한다.
- 사용자가 AI 추천을 검토하고 선택 적용하는 방식으로 볼트에 대한 통제권을 유지한다.
- 동시 요청 1개 기준으로 노트 100개당 처리 시간 60초 이내로 유지한다.

## 2. 사용자 페르소나 및 사용자 여정

### 페르소나 A: 연구자 김민준

- 직업: 대학원생, AI/ML 연구
- Obsidian 사용 기간: 2년
- 볼트 규모: 노트 약 800개
- 고민: 논문 메모, 아이디어 노트, 회의록이 뒤섞여 있고 태그가 일관성 없이 붙어 있다. 특정 주제로 작성한 노트를 다시 찾으려면 검색어를 여러 번 바꿔가며 시도해야 한다.
- 기대: 노트를 작성할 때마다 분류를 신경 쓰지 않아도 나중에 AI가 정리해 주길 원한다.

### 페르소나 B: 프리랜서 기획자 이서연

- 직업: UX 컨설턴트
- Obsidian 사용 기간: 1년
- 볼트 규모: 노트 약 300개
- 고민: 클라이언트별, 프로젝트별로 폴더를 만들었지만 경계가 모호한 노트가 많다. 태그를 붙이는 습관이 없어 나중에 참고 자료를 찾기 어렵다.
- 기대: 기존 노트에 소급 적용 가능한 자동 태그 추천과 간단한 질의 검색.

### 사용자 여정 1: 정리/분류 흐름

1. 사용자가 Obsidian을 열면 Librarian 사이드바 패널이 열려 있다.
2. 사용자가 "전체 볼트 스캔" 버튼을 클릭한다.
3. 플러그인이 볼트의 모든 마크다운 파일을 읽어 인덱스를 생성한다 (F001).
4. 각 노트에 대해 Claude API를 호출하여 1~2문장 요약, 태그 추천, 분류 근거를 동시에 생성한다 (F002). 요약은 NoteIndex에 저장되어 이후 검색에 재활용된다.
5. 사이드바 패널에 추천 목록이 표시된다: 노트 제목, 현재 태그, 추천 태그, 추천 이유 (F007).
6. 사용자가 추천 목록을 검토하며 개별 항목을 수락/거절하거나 전체 수락 버튼을 클릭한다 (F003).
7. 수락된 추천이 실제 노트 파일에 적용된다 (frontmatter tags 업데이트).
8. 완료 알림이 표시되고 적용 결과 요약이 나타난다.

### 사용자 여정 2: 검색 흐름

1. 사용자가 사이드바 패널의 검색 탭을 클릭한다.
2. 자연어 질의 입력창에 "React 상태 관리에 대해 고민한 노트"를 입력하고 Enter를 누른다 (F004).
3. 플러그인이 인덱스에 저장된 요약 데이터(전체 본문 아님)를 Claude API로 전송하여 의미적으로 관련된 노트를 검색한다. 토큰 사용량이 대폭 줄어 수천 개 볼트에서도 안정적으로 동작한다.
4. 검색 결과 목록이 관련도 순으로 표시된다: 노트 제목, 일치 이유, 요약 미리보기 (F005).
5. 사용자가 결과 항목을 클릭하면 해당 노트가 Obsidian 편집기에서 열린다.

## 3. 기능 명세

### Phase 1 MVP (F001~F007)

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 뷰/패널 |
|----|--------|------|-------------|------------|
| **F001** | 노트 스캔 및 인덱싱 | 볼트 전체 마크다운 파일을 읽어 제목, 본문, 기존 태그, 수정일을 인덱스에 저장하고 saveData로 영속화. 볼트 재시작 시 저장된 인덱스를 복원하여 변경된 파일만 재스캔. 본문은 AI 처리 전 임시 보유. | 모든 AI 기능의 데이터 기반 | 사이드바 패널 (분류 탭) |
| **F002** | AI 요약 및 분류/태그 추천 | 인덱싱된 노트 본문을 Claude API에 전송하여 1~2문장 요약, 태그 목록, 분류 근거를 응답으로 받아 NoteIndex의 summary 필드에 저장. 이 요약이 이후 검색의 핵심 데이터로 활용됨. API 응답 429(Rate Limit) 발생 시 지수 백오프로 최대 3회 재시도 (1초 → 2초 → 4초). | 핵심 가치: 자동 분류 + 검색 가능 인덱스 생성 | 사이드바 패널 (분류 탭) |
| **F003** | 사용자 검토 및 적용 | 추천 목록을 UI로 표시하고 사용자가 개별 또는 전체 수락/거절 후 노트 파일에 실제 적용 | 사용자 통제권 보장 필수 | 추천 검토 모달, 사이드바 패널 (분류 탭) |
| **F004** | 자연어 검색 쿼리 입력 | 사이드바 검색 탭의 텍스트 입력창에 자연어 질의를 입력하고 검색 요청을 Claude API로 전송. 검색 대상은 전체 본문이 아닌 NoteIndex의 summary 필드. | 핵심 가치: 토큰 효율적 의미 기반 검색 | 사이드바 패널 (검색 탭) |
| **F005** | 검색 결과 표시 | Claude API 응답을 파싱하여 관련 노트 목록을 관련도 순으로 표시하고 클릭 시 노트 오픈 | 검색 결과 제공 필수 | 사이드바 패널 (검색 탭) |
| **F006** | 플러그인 설정 | Claude API 키 입력(로컬 평문 저장), 모델 선택(기본값 claude-haiku-4-5-20251001), 스캔 대상 폴더 제외 설정 | 다양한 환경에서 동작하려면 설정 필수 | Obsidian 설정 탭 |
| **F007** | 사이드바 패널 UI | 분류 탭과 검색 탭으로 구성된 Obsidian 사이드바 뷰, 스캔 진행률 표시, 추천 목록 렌더링 | 모든 기능의 진입점 | 사이드바 패널 |

### Phase 2 Post-MVP (F101~F104)

| ID | 기능명 | 설명 | 구현 시점 이유 |
|----|--------|------|--------------|
| **F101** | 점진적 증분 분석 | 파일 변경 감지 이벤트(vault.on('modify'))를 통해 변경된 노트만 재분석하여 API 비용 절감. 변경된 노트의 요약도 자동 갱신. | MVP 이후 사용량 증가 시 비용 최적화 |
| **F102** | 검색 결과 순위/필터링 | 검색 결과에 날짜 범위, 태그, 폴더 기준 필터와 관련도 점수 정렬 추가 | 기본 검색 검증 후 고도화 |
| **F103** | 분류 규칙 커스터마이징 | 사용자 정의 태그 어휘, 폴더 구조 규칙을 설정에서 입력하면 AI 프롬프트에 반영 | 고급 사용자 요구 대응 |
| **F104** | 노트 링크 관계 분석 | 위키링크와 백링크를 분석하여 관련 노트 클러스터를 시각화하거나 추천에 반영 | 그래프 기능 의존성으로 후순위 |

### MVP 제외 기능

- 노트 자동 이동/이름 변경 (사용자 파일 통제권 침해 위험)
- 실시간 타이핑 중 자동 분석 (API 비용 과다)
- 로컬 임베딩 모델 지원 (초기 복잡도 과다)
- 클라우드 동기화 (Obsidian 철학과 상충)

## 4. UI/UX 흐름

### 4.1 사이드바 패널 흐름

```
Obsidian 실행
    |
    v
사이드바 패널 열림 (Librarian 아이콘 클릭 또는 자동)
    |
    +-- [분류 탭] ──────────────────────────────────────
    |       |
    |       v
    |   "전체 볼트 스캔" 버튼 클릭
    |       |
    |       v
    |   스캔 진행률 표시 (파일 N/전체 처리 중...)
    |       |
    |       v
    |   [F002] 각 노트별 요약 + 태그 추천 생성 (요약은 NoteIndex.summary에 저장)
    |       |
    |       v
    |   추천 목록 표시 (노트별 카드: 제목, 요약, 추천 태그, 이유)
    |       |
    |       +-- 개별 수락 버튼 → 해당 노트 frontmatter 업데이트
    |       +-- 개별 거절 버튼 → 목록에서 제거
    |       +-- "전체 수락" 버튼 → 모달 확인 후 일괄 적용
    |       +-- 카드 클릭 → 추천 검토 모달 (상세 이유, 편집 가능)
    |       |
    |       v
    |   완료 알림 (N개 노트에 태그 적용됨)
    |
    +-- [검색 탭] ──────────────────────────────────────
            |
            v
        자연어 질의 입력창 (placeholder: "예: 지난달에 정리한 독서 노트")
            |
            v
        Enter 또는 검색 버튼 클릭
            |
            v
        [인덱스 확인] 요약 데이터 있음? 없으면 "먼저 볼트를 스캔해 주세요" 안내
            |
            v
        로딩 스피너 표시 (Claude API 처리 중)
        (전송 데이터: 질의 텍스트 + NoteIndex.summary 배열, 전체 본문 제외)
            |
            v
        검색 결과 목록 표시 (관련도 순)
            |
            v
        결과 항목 클릭 → 해당 노트 오픈
```

### 4.2 추천 검토 모달

- 트리거: 분류 탭에서 노트 카드 클릭
- 내용: 노트 제목, AI 생성 요약 (1~2문장), 본문 미리보기 (첫 200자), 현재 태그, 추천 태그 (편집 가능 칩 형태), 추천 이유 텍스트
- 액션 버튼: "적용", "취소"

### 4.3 Obsidian 설정 탭

- 진입: Obsidian 설정 > Librarian
- 항목:
  - Claude API 키 (비밀번호 형태 입력, **로컬 평문 저장** - Obsidian Plugin API의 data.json에 일반 텍스트로 저장됨을 UI에서 고지)
  - 모델 선택 드롭다운 (claude-haiku-4-5-20251001 기본값, claude-sonnet-4-6, claude-opus-4-6 선택 가능)
  - 스캔 제외 폴더 (쉼표 구분 텍스트 입력)
  - 최대 동시 API 요청 수 (1~3 슬라이더, 기본값 1)

## 6. 데이터 모델

### NoteIndex (노트 인덱스)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| path | 볼트 내 파일 경로 (고유 키) | string |
| title | 노트 제목 (파일명 또는 첫 H1) | string |
| content | 마크다운 본문 전체 텍스트 (AI 처리 후 메모리에서 해제 가능) | string |
| **summary** | **AI가 생성한 1~2문장 노트 요약 (F002 처리 후 저장, 검색 시 핵심 데이터로 활용)** | **string** |
| existingTags | 현재 frontmatter의 태그 배열 | string[] |
| lastModified | 파일 마지막 수정 시각 | number (Unix timestamp) |
| indexedAt | 인덱싱 처리 완료 시각 | number (Unix timestamp) |

### ClassificationRecommendation (분류 추천)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| notePath | 대상 노트 경로 | string -> NoteIndex.path |
| recommendedTags | AI가 추천한 태그 배열 | string[] |
| reason | 추천 근거 텍스트 (Claude 응답) | string |
| status | 검토 상태 | 'pending' / 'accepted' / 'rejected' |
| createdAt | 추천 생성 시각 | number (Unix timestamp) |

### SearchResult (검색 결과)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| notePath | 결과 노트 경로 | string -> NoteIndex.path |
| title | 노트 제목 | string |
| relevanceExplanation | AI가 생성한 관련도 설명 텍스트 | string |
| summaryExcerpt | NoteIndex.summary 기반 미리보기 텍스트 (전체 본문 아님) | string |
| rank | 결과 내 순위 (1부터 시작) | number |

| 필드 | 설명 | 타입/관계 |
|------|------|----------|

| maxConcurrentRequests | 동시 API 요청 최대 수 (기본값 1) | number (1~3) |

## 7. 기술 스택

### 플러그인 코어

- **TypeScript 5.6+**: 타입 안전성 보장, Obsidian Plugin API와 완전한 타입 통합
- **Obsidian Plugin API**: 볼트 파일 시스템 접근(vault.getFiles, vault.read, vault.modify), 사이드바 뷰(ItemView), 설정 탭(PluginSettingTab), 모달(Modal), 이벤트 리스닝(vault.on)
- **esbuild**: Obsidian 플러그인 표준 번들러, 빠른 빌드

### AI 통합

- **Anthropic Claude API (REST)**: 노트 요약 생성, 분류/태그 추천 및 자연어 검색 의미 분석
  - 기본 모델: `claude-haiku-4-5-20251001` (속도/비용 최적화)
  - 선택 가능 모델: `claude-sonnet-4-6` (균형), `claude-opus-4-6` (최고 품질)
- **Obsidian requestUrl() API**: Claude REST API 직접 호출에 사용. **Node.js fetch 대신 반드시 Obsidian의 requestUrl()을 사용해야 함** - Obsidian 플러그인 환경에서 fetch는 CORS 제한으로 외부 API 호출이 차단되거나 동작이 불안정할 수 있음.

### UI

- **Obsidian 기본 UI 컴포넌트**: PluginSettingTab, Modal, Notice (토스트 알림)
- **순수 TypeScript DOM 조작**: 사이드바 패널 뷰는 ItemView.containerEl에 직접 HTML 생성
- **CSS (Obsidian 테마 변수 활용)**: var(--background-primary), var(--text-normal) 등 테마 대응

### 데이터 저장

- **Obsidian Plugin API saveData/loadData**: PluginSettings를 볼트 내 .obsidian/plugins/obsidian-librarian/data.json에 저장 (API 키 포함 평문 저장, 암호화 없음)
- **인덱스 영속화**: NoteIndex 배열(summary 포함)을 saveData/loadData로 data.json에 함께 저장. 볼트 재시작 시 저장된 인덱스를 먼저 복원하고 lastModified 비교로 변경된 노트만 재스캔하여 API 비용과 대기 시간 최소화.
- **ClassificationRecommendation 임시 저장**: 스캔 세션 중 메모리 유지, 모두 처리 후 소멸

### 개발 도구

- **Node.js 20+**: 개발 환경
- **npm**: 패키지 관리
- **obsidian-typings**: Obsidian API 타입 정의
- **ESLint + Prettier**: 코드 품질 관리

## 8. 성공 지표

### 정확도 지표

- **태그 추천 수락률**: 사용자가 AI 추천 태그를 수정 없이 수락하는 비율 목표 70% 이상
- **검색 관련도**: 검색 결과 상위 3개 중 사용자가 원하는 노트가 포함되는 비율 목표 85% 이상

### 성능 지표

- **전체 스캔 속도**: 노트 100개당 처리 시간 60초 이내 (API 응답 시간 포함, 동시 요청 1개 기준)
- **검색 응답 시간**: 자연어 질의 입력 후 결과 표시까지 5초 이내
- **검색 토큰 사용량**: 볼트 1,000개 기준 검색 1회당 요약 텍스트 총량이 Claude 컨텍스트 한계(200K 토큰) 이내로 유지

### 사용자 경험 지표

- **작업 완료율**: 스캔 시작 후 추천 검토 완료까지 이탈 없이 완료하는 비율 목표 80% 이상
- **재사용률**: 주 1회 이상 플러그인을 능동적으로 사용하는 사용자 비율 목표 60% 이상

### API 비용 지표

- **노트 1개당 평균 API 비용**: claude-haiku-4-5-20251001 기준 $0.001 이하 유지
- **검색 1회당 평균 API 비용**: 800개 노트 기준 약 $0.03~$0.04 (요약 24K~32K 토큰 입력 기준). Prompt Caching 적용 시 $0.005 이하 달성 가능 (Post-MVP 적용 예정)
